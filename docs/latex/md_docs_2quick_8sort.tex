\hypertarget{md_docs_2quick_8sort_autotoc_md49}{}\doxysection{\texorpdfstring{Алгоритм работы}{Алгоритм работы}}\label{md_docs_2quick_8sort_autotoc_md49}

\begin{DoxyEnumerate}
\item Выбирается опорный элемент (pivot), который делит массив на две части.
\item Все элементы меньше опорного перемещаются в левую часть, а все элементы больше — в правую часть (если сортировка по возрастанию).
\item Рекурсивно сортируются обе части массива.
\item Процесс продолжается до тех пор, пока каждая часть не будет отсортирована.
\end{DoxyEnumerate}\hypertarget{md_docs_2quick_8sort_autotoc_md50}{}\doxysection{\texorpdfstring{Примечения}{Примечения}}\label{md_docs_2quick_8sort_autotoc_md50}

\begin{DoxyItemize}
\item В качестве опорного элемента (pivot) может быть выбран не только последний элемент в текущем подмассиве, но и любой другой элемент.
\item Алгоритм quick\+Sort имеет временную сложность O(n log n) в среднем и O(n²) в худшем случае, но в большинстве случаев работает быстрее пузырьковой сортировки.
\item Оптимально использовать этот алгоритм для сортировки больших объемов данных, однако в определенных ситуациях его эффективность может снизиться. 
\end{DoxyItemize}
